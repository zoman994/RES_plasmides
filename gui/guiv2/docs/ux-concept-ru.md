# UX/UI концепция PlasmidVCS: анализ лучших практик и дизайн-решения

**Суть в одном абзаце:** PlasmidVCS должен соединить визуализацию плазмид уровня SnapGene, граф ветвлений из GitKraken, миниатюрное версионирование из Figma и цветовое отслеживание авторства из Google Docs — в одном React-приложении, построенном поверх Open Vector Editor (MIT-лицензия). Ни один существующий инструмент не объединяет семантический diff ДНК-конструктов с визуальным контролем версий. Это и есть незанятая ниша.

Целевой пользователь — инженер штаммов в лаборатории на 3–20 человек, который ведёт 10–50 конструктов, редактирует последовательности в SnapGene, а историю и связи между версиями отслеживает через Excel + папки с файлами.

## Часть 1. Как лучшие редакторы плазмид визуализируют ДНК

### SnapGene — золотой стандарт с тремя синхронными видами

Центральный просмотрщик показывает **кольцевую карту**: features — цветные направленные дуги вокруг кольца, позиция +1 закреплена сверху. Стрелки показывают направление (по часовой = прямая цепь, против = обратная). Сайты рестрикции — радиальные засечки внутрь кольца. Названия features — снаружи кольца, на выносных линиях, с автоматическим разрешением коллизий.

**Линейная карта** разворачивает те же данные в горизонтальные полосы над и под линией backbone. Три точки («...») на краях обозначают, что кольцевая последовательность оборачивается. Переключение между видами — мгновенное, одной кнопкой.

**Sequence view** — нуклеотидный текст с цветной подсветкой, совпадающей с цветами карты. Обе цепи. Трансляция ORF инлайн. Сверху — **minimap** со сжатой полноразмерной последовательностью и выделенной областью просмотра.

Убийственная фича SnapGene — **History View**. Графическое дерево каждой клонирующей операции (ПЦР, Gibson, рестрикция, лигирование), которая привела к текущему конструкту. Каждый узел кликабелен — предковые последовательности встроены в сам файл. «History Colors» на карте подсвечивают, какие регионы откуда пришли. Это ближайший существующий аналог того, что нужно PlasmidVCS. Но он линейный (нет ветвления), файловый (нет облака) и однопользовательский.

Автодетекция features из курируемой базы — ещё одна сильная сторона SnapGene. Пользователи любят качество карт и симуляцию клонирования. Главные жалобы: **цена $149–995/год** и **нулевые возможности коллаборации** — десктоп, без облака, шеринг только через email .dna-файлов.

### Benchling — коллаборация есть, глубины нет

Веб-интерфейс Benchling организован вокруг трёх вкладок на последовательность: **Sequence Map** (нуклеотидный текст с аннотациями), **Linear Map** (горизонтальный вид), **Plasmid Map** (кольцевой). Все три синхронизированы — выделение в одном подсвечивает в остальных. Шестерёнка справа вверху управляет видимостью аннотаций, праймеров, сайтов рестрикции.

Модель версий Benchling логирует каждое изменение с таймстемпом и авторством. Есть «Reset to version». Аудит-логи обеспечивают регуляторную прослеживаемость. Коллаборация реальновременная с READ/WRITE/ADMIN правами. Последовательности можно заблокировать от случайных правок.

Главная слабость: **анализ последовательностей слабее SnapGene**. Benchling не визуализирует историю конструкта как дерево. Визуального diff между версиями нет — только список правок с таймстемпами.

### Geneious — лучший alignment viewer

Geneious ввёл паттерн **Circular Overview**: маленькая кольцевая карта рядом с увеличенным линейным видом, зелёный прямоугольник показывает текущую видимую область. Двойной вид даёт постоянный пространственный контекст при работе на уровне пар оснований.

Alignment viewer — самый мощный среди редакторов плазмид. Выровненные последовательности идут рядами, консенсус сверху. **Identity graph** окрашивает колонки: зелёный = 100% идентичность, жёлтый = частичная, красный = низкая. **Coverage graph** — столбики: синий (мин), чёрный (среднее), жёлтый (макс). **Dot Plot** строит совпадения двух последовательностей на осях X/Y: диагонали = контигные совпадения, параллельные линии = повторы, разрывы = инделы.

Система цветов самая детальная: отдельные схемы для идентичности нуклеотидов, качества base-call (оттенки синего), расстояния paired-read (зелёный = корректно, красный = неправильная ориентация), направления чтения, качества маппинга — все настраиваемые побазово.

### ApE — минимализм работает

ApE — бесплатное Tcl/Tk приложение — доказывает, что лёгкий инструмент покрывает 80% задач. Features — цветные фоны на тексте последовательности. Вертикальный **feature minimap** справа показывает все features по всей последовательности. Кольцевая карта рендерится в отдельном окне — функционально, но не так красиво, как в SnapGene.

Урок для PlasmidVCS: пользователи терпят простой UI, если он **бесплатный, быстрый и надёжный**. ApE живёт 15+ лет именно потому, что запускается мгновенно, работает на чём угодно и не падает.

### Asimov Kernel — GitHub-для-ДНК

Kernel ближе всего к тому, чем хочет стать PlasmidVCS. Ядро — **организация по репозиториям**: конструкты живут в repo с workspace'ами, правами и коллаборацией. Дизайн-канвас позволяет drag-and-drop частей из поисковой библиотеки **650 000+ ДНК-последовательностей** (включая полный iGEM Registry и Addgene). Три формата отображения: линейная последовательность, кольцевая карта, интерактивная схема.

Asimov прямо говорят, что добавляют version control. Их блог-пост «GitHub is to Code as ___ is to DNA» формулирует ровно ту же идею: репозитории, форки, отслеживание изменений, деревья происхождения.

Что взять: репозитории с гранулярными правами, drag-and-drop сборка из библиотек, несколько режимов отображения. Чего у них нет для нас: Kernel заточен под enterprise-фарму (CHO, антитела, AI-оптимизация кодонов) — для лаборатории на 5 человек это избыточно.

### Open Vector Editor — open-source фундамент

**Open Vector Editor (OVE)** от TeselaGen — MIT-лицензионный React + Redux компонент, рендерящий кольцевые и линейные карты плазмид через SVG. Это самая релевантная стартовая точка для фронтенда PlasmidVCS.

Архитектура: **настраиваемый многопанельный layout** с Circular Map, Linear Map (rail view), Sequence Map (текст) и Properties — в два столбца с вкладками. SVG-рендеринг рисует features как цветные дуги со стрелками-направлениями на концентрических кольцах, с автоматическим размещением подписей через leader lines.

Модель данных чистая: последовательности — JSON-объекты с `sequence` (строка), `circular` (boolean), `features` (массив `{color, type, start, end, id, forward}`). Индексация **0-based inclusive** — удобно для кольцевой арифметики. Библиотеки-компаньоны: `@teselagen/bio-parsers` (парсинг GenBank, FASTA, SnapGene, Geneious) и `@teselagen/sequence-utils` (манипуляция последовательностями).

Существует `SimpleCircularOrLinearView` — облегчённый компонент для read-only встраивания без Redux. Идеален для миниатюр в истории версий.

## Часть 2. Паттерны UI контроля версий, переносимые на биологию

### GitHub — словарь визуализации изменений

**Unified view** — старое и новое чередуются в одном столбце: красный фон (#ffebe9) для удалений, зелёный (#e6ffec) для добавлений. **Split view** — оригинал слева, изменённое справа. Критически для ДНК: **посимвольная подсветка** использует более тёмные оттенки (#fdb8c0 для удалённых символов, #acf2bd для добавленных) внутри изменённых строк — чтобы заметить однонуклеотидные мутации в сохранённом регионе.

**Сворачивание неизменённых регионов** — ключевое для плазмид: в конструкте на 6 кб, где изменились 3 основания, надо скрыть 5997 идентичных оснований и подсветить 3 мутации.

**Inline comments** — клик на `+` рядом с любой строкой начинает threaded-обсуждение. Прямо переносится на аннотации: «Почему сменили этот кодон?» — привязано к конкретной позиции.

**Blame view** — каждая строка аннотирована SHA коммита, аватаром автора и датой, сгруппированные по коммитам с цветными бордюрами слева. Может стать «кто последний менял этот регион плазмиды».

### GitKraken — модель для генеалогии конструктов

Граф коммитов GitKraken — самый переносимый UI-паттерн для истории конструктов. Центральная панель рисует DAG вертикально: **каждая строка = коммит, каждый столбец/дорожка = ветка** в виде цветной вертикальной линии. Слияния — сходящиеся линии. Метки веток — цветные пилюли у HEAD-коммитов.

Ключевые интеракции для адаптации:

- **Hover-to-highlight**: наведение на метку ветки подсвечивает все коммиты этой ветки, остальные затеняет — идеально для «какие конструкты произошли от P43_Cas v3?»
- **Drag-and-drop merge**: перетаскивание одной ветки на другую = merge — у нас это «объединить элементы двух вариантов конструкта»
- **WIP node**: верхний узел = незакоммиченные изменения — аналог «правки в процессе»
- **Цветовые дорожки**: каждый вариант конструкта получает свой яркий цвет, сохраняющийся по всему графу

Для PlasmidVCS каждая «ветка» — вариант конструкта (например, «с-GFP-тегом», «кодон-оптимизированный-для-CHO»). Каждый «коммит» — версионированный снимок конструкта. Merge — где элементы двух вариантов объединились.

### VS Code — гаттерные индикаторы для разметки изменений

VS Code ставит цветные полоски в левом гаттере: **зелёная = добавленные строки, синяя = изменённые, красный треугольник = удалённые**. Эти же индикаторы видны на **minimap** (обзор кода рядом со скроллбаром), давая bird's-eye view распределения изменений.

Для линейного вида последовательностей PlasmidVCS: гаттерные индикаторы отмечают, какие bp-регионы добавлены, изменены или удалены по сравнению с предыдущей версией. Minimap — сжатое представление всей плазмиды рядом со скроллбаром — покажет цветные точки концентрации изменений. На кольцевой карте аналог — цветные сегменты на внутреннем кольце.

### Figma — миниатюрное версионирование для визуального контента

История версий Figma: **автосохранение + именованные версии**: автоматические чекпоинты каждые 30 минут + пользовательские milestones с заголовком (25 символов) и описанием (~140 символов). Сайдбар показывает хронологический список; клик на любую версию рендерит **полный визуальный снимок**.

Для плазмид — мощный паттерн: каждая версия в сайдбаре показывает **миниатюру кольцевой карты** конструкта на тот момент — мгновенно узнаваемую по раскладке features. Именованные версии вроде «Добавлен GFP-репортер» или «Замена AmpR → KanR» — человекочитаемые milestones.

### Google Docs — цветовое авторство для мультипользовательского трекинга

Google Docs назначает каждому соавтору свой цвет. В истории версий **добавления подсвечиваются цветом автора; удаления — зачёркиванием**. Режим «Suggesting» показывает предложенные изменения зелёным подчёркиванием (вставки) и цветным зачёркиванием (удаления), с кнопками Accept (✓) / Reject (✗).

Для PlasmidVCS: когда несколько исследователей работают с конструктом, изменения каждого — в его цвете и на последовательности, и на карте. «Suggesting mode» для предложенных замен кодонов — прорывная фича для совместного дизайна конструктов.

## Часть 3. Конвенции визуализации из биоинформатики

### Alignment viewers задают грамматику отображения различий

**IGV** ввёл **семантический зум**: на масштабе хромосомы данные — столбиковые графики покрытия; на ~50 кб видны отдельные риды как серые полоски; при однобазовом разрешении мисматчи — цветные буквы с прозрачностью пропорциональной качеству base-call. **Вставки** — фиолетовые блоки; **делеции** — чёрные горизонтальные линии. Мультирезолюционный подход необходим VCS-инструменту — плазмида на 10 кб при полном зуме показывает каждое основание, а в обзоре — feature-уровневую структуру.

**JBrowse 2** добавляет плавную анимированную навигацию (pan и zoom в стиле Google Maps), rubberband selection для zoom-to-region и двойную шкалу масштаба. Самый переносимый паттерн — **поведение при разных уровнях зума**: при удалении генные предикции — гистограммы плотности; при приближении — интрон/экзон структуры с разной толщиной для UTR и кодирующих экзонов.

**UCSC Genome Browser** — пять уровней плотности на трек: Hide, Dense, Squish, Pack, Full. Для PlasmidVCS нужны минимум три уровня аннотаций: collapsed (цветные сегменты на backbone), packed (подписанные стрелки), full (детально с суб-features).

### Де-факто стандарт цветов для features плазмид

Формального стандарта нет, но доминирование SnapGene (используется Addgene для 170 000+ карт плазмид) создаёт устойчивые конвенции:

| Тип feature | Цвет | Hex | Символ |
|---|---|---|---|
| **CDS / кодирующая область** | Оранжевый / янтарный | `#F5A623` | Направленная стрелка |
| **Промотор** | Белый / светло-серый | `#E0E0E0` | Изогнутая стрелка |
| **Терминатор** | Красный / тёмно-красный | `#CC0000` | Т-бар / леденец |
| **Ориджин репликации** | Жёлтый / золотой | `#FFD700` | Залитая стрелка или дуга |
| **Маркер селекции (AbR)** | Зелёный | `#31AF31` | Большая направленная стрелка |
| **Репортерный ген (GFP)** | Циан / ярко-зелёный | `#00CC66` | Направленная стрелка |
| **Регуляторные элементы** | Голубой | `#6699CC` | Короткий блок или стрелка |
| **Прочее** | Серый | `#999999` | Прямоугольник |

Для нуклеотидных оснований стандарт IGV: **A = зелёный (#00CC00), C = синий (#0000CC), G = жёлтый (#FFD700), T = красный (#CC0000)**.

### Синхронизация кольцевого и линейного видов — обязательна

Каждый серьёзный инструмент синхронизирует кольцевую карту, линейную карту и текст последовательности. Минимум: клик на feature в любом виде подсвечивает во всех; выделение региона на кольцевой карте скроллит текстовый вид; позиция ориджина (+1) настраиваемая и всегда показана сверху кольцевой карты. **Circular Overview** из Geneious (маленькая кольцевая карта с зелёным прямоугольником текущего viewport) — самое элегантное решение для сохранения контекста при работе на уровне пар оснований.

## Часть 4. Детальные дизайн-рекомендации для PlasmidVCS

### 4.1 Визуализация карты плазмиды: двойная архитектура

**Технология:** SVG-рендеринг через React (архитектура Open Vector Editor) с `@teselagen/bio-parsers` для импорта GenBank/FASTA/SnapGene. Внутреннее хранение — JSON, индексация 0-based inclusive.

**Кольцевая карта (основной вид):**

- Features — цветные дуги со стрелками-направлениями, цвета по таблице выше
- Несколько концентрических колец: внешнее для features, среднее для частей, внутреннее для сайтов рестрикции
- Позиция +1 всегда на 12 часов; переиндексация правым кликом на любую позицию
- Подписи — снаружи кольца на выносных линиях; автодетекция коллизий
- Тонкое внутреннее кольцо — backbone с метками масштаба в bp
- Выделение — подсвеченный сегмент дуги с ручками на концах
- **Зум**: выделение региона → «Focus on Region» раскрывает дугу. Колесо мыши — плавный зум
- **Mini circular overview** (паттерн Geneious): в линейном или текстовом виде маленькая кольцевая карта (150×150 px) с зелёным прямоугольником текущего viewport — всегда видна

**Линейная карта (вторичный вид):**

- Горизонтальная backbone-линия, features — цветные направленные полоски сверху (+ цепь) и снизу (– цепь)
- Горизонтальный скроллинг; wrap для полной последовательности
- Три точки на концах для кольцевых последовательностей
- Цвета согласованы с кольцевым видом

**Текстовый вид (вид редактирования):**

- Моноширинный нуклеотидный текст строками по 60–80 bp
- Features — цветная подсветка фона, совпадающая с картой
- Обе цепи, трансляция ORF
- Сайты рестрикции — перевёрнутые галки над позицией реза
- Гаттер слева с номерами позиций
- **Индикаторы изменений** (паттерн VS Code): цветные полоски в гаттере — зелёная для добавленных оснований, синяя для изменённых, красный треугольник для удалённых — относительно предыдущей версии

**Все три вида синхронизированы**: выделение в одном мгновенно подсвечивает в остальных. Горячая клавиша для переключения (Ctrl+1/2/3).

### 4.2 Diff-вид для генетических конструктов: два семантических уровня

Самая новая фича PlasmidVCS. Два слоя визуализации:

**Уровень 1 — семантический diff на уровне features (вид по умолчанию):**

- Кольцевые карты Version A (слева) и Version B (справа) рядом
- Общие/неизменённые features — в обычных цветах на тех же угловых позициях
- **Добавленные features** (есть в B, нет в A): ярко-зелёные с бейджем `+`
- **Удалённые features** (есть в A, нет в B): красные с зачёркнутым именем и бейджем `−`
- **Изменённые features** (то же имя/тип, другая последовательность): янтарные/жёлтые с бейджем `~`
- **Журнал изменений** под картами: структурированный список — «Промотор CMV → EF1α (1204 bp → 1182 bp)», «Добавлено: 6×His-тег на позиции 3420», «Удалён: сайт BamHI на позиции 892»
- **Blink toggle** (паттерн Kaleidoscope): кнопка для быстрого чередования двух версий на одной карте — различия «мигают» и становятся очевидными

**Уровень 2 — побазовый diff (drill-down):**

- Клик по изменённому feature или региону раскрывает базовое выравнивание
- **Split view** (GitHub-стиль): старая последовательность слева, новая справа, посимвольная подсветка зелёным/красным
- **Unified view** — переключатель: чередование строк с `-` (удалено, красный) и `+` (добавлено, зелёный)
- **Сворачивание неизменённых регионов**: идентичные основания свёрнуты — «Показать 2340 идентичных оснований»
- Внутри изменённых CDS — аннотация миссенс-мутаций: «A→G на позиции 1523 — Lys→Arg»
- Inline comments привязаны к конкретным позициям, threaded-обсуждения

**Палитра diff-цветов:**

- Добавления: фон `#e6ffec`, символы `#acf2bd` (зелёная гамма GitHub)
- Удаления: фон `#ffebe9`, символы `#fdb8c0` (красная гамма GitHub)
- Модификации: фон `#fff8e1`, символы `#ffe082` (янтарный)
- Без изменений: прозрачный

### 4.3 Таймлайн истории конструкта с миниатюрами

Комбинация графа GitKraken и миниатюр Figma:

**Layout**: левый сайдбар (200 px), вертикальный таймлайн версий. Каждая запись содержит:

1. **Миниатюра кольцевой карты** (80×80 px, через `SimpleCircularOrLinearView`) — мгновенно узнаваемый layout features
2. **Имя версии** (пользовательское: «Добавлен GFP-репортер») или автогенерированное («3 features изменены»)
3. **Аватар + имя автора** с цветовым индикатором
4. **Таймстемп** (относительный: «3 дня назад»)
5. **Бейдж изменений**: «+2 features, −1 feature, ~340 bp модифицировано»

**Интеракции:**

- Клик на версию — загрузить в основной просмотрщик
- Shift+клик на две версии — открыть diff между ними
- Правый клик → «Создать вариант от этой версии» — ветвление
- Drag для сравнения: перетащить миниатюру одной версии на другую — запуск diff
- **Автосохранение + именованные версии** (модель Figma): каждая правка создаёт автоматический чекпоинт; пользователь явно именует milestones кнопкой «Сохранить версию»

**Визуализация веток** (когда конструкт разветвился):

- Таймлайн превращается из линейного списка в **мини-граф веток** (стиль GitKraken)
- Каждый вариант — своя цветная дорожка
- Точки ветвления показывают, где вариант отделился
- Точки слияния — где изменения были объединены

### 4.4 Линейдж штаммов — интерактивный DAG

Визуализация связей parent→child между штаммами.

**Layout**: горизонтальный направленный ациклический граф (DAG), слева направо:

- Каждый **узел** — штамм, карточка (180×100 px):
  - Имя штамма (жирное: «AN-004»)
  - Миниатюра кольцевой карты конструкта (50×50 px)
  - Иконка/надпись организма-хозяина
  - Дата создания
  - Бейдж статуса: 🟢 Активный / 🟡 В работе / 🔴 Списан / ⚪ Архив
- **Рёбра** — направленные стрелки с подписью метода трансформации («PEG-протопласт», «электропорация»)
- **Тултипы** при наведении на ребро: дата, оператор, конструкт, метод селекции

**Интеракции:**

- Клик по узлу — расширить панель деталей (правый сайдбар): полные метаданные, связанные конструкты, экспериментальные заметки
- Двойной клик — перейти к конструкту этого штамма в редакторе
- Фильтры: по дате, организму, backbone, статусу
- Зум: колесо мыши; кнопка «вместить всё»
- Цветовая кодировка узлов по проекту, организму или исследователю
- Layout через dagre.js или ELK для автоматической компоновки

### 4.5 Браузер библиотеки частей — карточки и умный поиск

**Layout**: панель (ресайзабельная, dock-left или оверлей):

**Поисковая строка** сверху:

- Свободнотекстовый поиск по именам, описаниям, последовательностям
- Чипы фильтров: тип (CDS, промотор, терминатор, ориджин, маркер, тег, линкер), источник (коллекция лаборатории, Addgene, iGEM Registry), совместимость с организмом
- BLAST-поиск: вставить последовательность, найти похожие части

**Карточки частей** в адаптивной сетке (3–4 столбца):

- Каждая карточка (160×120 px):
  - Имя (жирное, truncated с тултипом)
  - Бейдж типа с иконкой (цветной кружок по конвенции: оранжевый для CDS, красный для терминатора и т.д.)
  - Длина (напр., «1204 bp»)
  - Мини-линейная карта — тонкая цветная полоска features
  - Счётчик использования («Используется в 7 конструктах»)
  - Иконка источника (🏠 Лаб, лого Addgene, лого iGEM)

**Интеракции:**

- Клик по карточке — развернуть детальный вид: полная последовательность, аннотации, история использования
- **Drag-and-drop** из библиотеки на канвас конструкта — вставка части (паттерн Kernel)
- Переключение на табличный вид для плотного сканирования (столбцы: имя, тип, длина, источник, последнее использование, число использований — сортируемые)

### 4.6 Дашборд — командный центр проекта

Главный вид проекта — overview для инженера штаммов.

**Верхняя панель**: имя проекта + breadcrumb, глобальный поиск, аватар, уведомления.

**Левая навигация** (узкая, иконная, раскрывающаяся):

- 🏠 Дашборд
- 🧬 Конструкты (список всех конструктов проекта)
- 🔬 Штаммы (линейдж-вид)
- 📦 Библиотека частей
- 📋 Лог активности
- ⚙️ Настройки

**Контентная область (четыре квадранта):**

**Верхний левый: последние конструкты** — горизонтальный скроллируемый ряд карточек (как в Netflix). Каждая: миниатюра кольцевой карты, имя, дата, аватар автора, бейдж числа версий. Карточка «Новый конструкт» с `+` в начале.

**Верхний правый: обзор линейджа штаммов** — миниатюрный DAG верхнего уровня. Клик «Показать полный линейдж» для раскрытия. Счётчики: активные штаммы, конструкты, последние деривации.

**Нижний левый: лента активности** — хронологический список событий: «Игорь создал конструкт pEXP-XynTL-v4» (с миниатюрой), «Середа изменил штамм AN-006» (+2 мутации), «Денисенко добавил часть TtrpC в библиотеку». Каждая запись кликабельна. Цветные левые бордюры по автору (паттерн Google Docs).

**Нижний правый: статистика + быстрые действия** — карточки: общее число конструктов (со спарклайном), штаммов, ожидающих ревью. Кнопки: «Импорт GenBank», «Создать конструкт», «BLAST-поиск».

**Визуальный стиль**: чистый, светлый фон, тонкие серые бордюры. 16 px базовый шрифт, Inter или IBM Plex Sans. Акцентный цвет — профессиональный сине-бирюзовый (`#0066CC` или `#00897B`). Карточки с `border-radius: 8px`, лёгкие тени. Тёмная тема по переключателю. Любая фича — не более 3 кликов от дашборда.

## Заключение: место PlasmidVCS на рынке

Экосистема инструментов молекулярной биологии имеет чёткий gap. SnapGene владеет качеством визуализации, но лишён коллаборации и контроля версий. Benchling владеет коллаборацией, но трактует версионирование как плоский аудит-лог, а не визуальную генеалогию конструктов. Asimov Kernel строит GitHub-для-ДНК, но целится в enterprise-фарму, а не в малые биотех-лаборатории.

**Ни один существующий инструмент не объединяет семантический diff конструктов с визуальными графами ветвлений и лёгким self-hosted развёртыванием.**

Конкурентная позиция PlasmidVCS опирается на три новых UI-вклада: **семантические diff на уровне features** (показывает «замена промотора», а не «1204 bp изменены»), **граф генеалогии конструктов в стиле GitKraken** с миниатюрами кольцевых карт в каждом узле, и **цветовой suggesting mode в стиле Google Docs** для предложенных изменений последовательностей.

Приоритет реализации: сначала кольцевая карта + diff-вид (ядро ценности), затем линейдж штаммов и библиотека частей, в конце — дашборд. Импорт/экспорт GenBank — table stakes, должен работать безупречно с первого дня.
