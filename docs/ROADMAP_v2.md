## Модуль 2: Импорт геномов

**Проблема:** Чтобы клонировать ген из E. coli, студент вручную ищет на NCBI, копирует последовательность. 10 минут на ген.
**Решение:** Загрузить аннотированный геном → парсинг ВСЕХ CDS → таблица → поиск → в библиотеку.

Backend: POST /api/genomes/import (Biopython SeqIO, ~3 сек для E. coli K-12)
Frontend: GenomeImporter.jsx — таблица генов, поиск по gene/product/EC, bulk add.

Каждый импортированный CDS получает: gene name, product, EC_number, locus_tag, sequence, protein translation, exon count, organism.

Тестовые геномы:
- E. coli K-12 MG1655 (4,240 CDS)
- E. coli BL21(DE3)
- A. niger CBS 513.88 (~11,000 CDS)

**Оценка: 1-2 дня**

## Модуль 3: EC → метаболические пути (статический маппинг)

**Проблема:** 4,000 генов — как найти нужный без контекста?
**Решение:** Словарь EC → pathway name. Без API, офлайн.

```python
EC_TO_PATHWAY = {
    "2.7.1.1":  "Гликолиз (гексокиназа)",
    "3.2.1.3":  "Деградация крахмала (глюкоамилаза)",
    "2.5.1.29": "Биосинтез терпеноидов (GGPP-синтаза)",
    # ~200 записей
}
```

Фильтр "Метаболический путь" в GenomeImporter.

**Оценка: 0.5 дня**

## Модуль 5: Авто-генерация экспрессионной кассеты

**Проблема:** Выбрал ген — нужно вручную подбирать промотор, RBS, терминатор.
**Решение:** Шаблоны кассет по организму-хозяину.

Шаблоны:
- E. coli: T7 promoter → B0034 RBS → [ген] → rrnB T1 terminator

Кнопка "Создать кассету" → все части на canvas → Generate Primers → готово.

## Модуль 6: CRISPR нокаут/интеграция за 3 клика

**Проблема:** Подбор gRNA → отдельный сайт. Плечи → вручную из генома. Сборка донора → вручную. Итого 2-3 часа.
**Решение:** Wizard: выбрал ген → gRNA подобран → плечи вырезаны → 2 сборки на canvas.

### Шаг 1: Выбор гена-мишени
Из импортированного генома (Модуль 2). Выпадающий список генов.

### Шаг 2: Подбор gRNA (guide_design.py, ~150 строк)

Сканирование гена на PAM-сайты. Scoring:
- GC content 40-70%: +20 баллов
- Нет poly-T ≥4 (терминация U6): обязательно
- G на позиции 20 (у PAM): +5
- Нет шпильки: обязательно
- Позиция в первой трети гена: +10 (для нокаута)
- Off-target: подсчёт ≤3 мисматчей + PAM в геноме

Нуклеазы:
- SpCas9 (PAM: NGG) — стандарт
- AsCpf1/Cas12a (PAM: TTTV) — уже есть в библиотеке!
- SaCas9 (PAM: NNGRRT) — компактный

UI: таблица топ-5 gRNA с score, GC%, позицией на схеме гена.

### Шаг 3: Стратегия и плечи (homology_arms.py, ~100 строк)

Три стратегии:
- **Нокаут:** [5'плечо] → [маркер pyrG/hygR] → [3'плечо]
- **Нокин:** [5'плечо] → [экспрессионная кассета] → [3'плечо]
- **Точечная мутация:** [5'плечо с мутацией] → [3'плечо]

Длина плечей: 500 / 750 / 1000 / 1500 п.н. (слайдер).

### Результат: 2 сборки автоматически

Сборка 1: gRNA кассета
  [PU6] → [spacer 20bp] → [sgRNA scaffold 76bp]

Сборка 2: Донорный конструкт
  [5'плечо 1kb] → [PgpdA→pyrG→TtrpC] → [3'плечо 1kb]
  Метод: Overlap PCR или Gibson

Обе сборки → Generate Primers → заказ олигов.

### Frontend: KnockoutWizard.jsx (~350 строк)
Доступ: кнопка "🎯 Нокаут" в header, или правый клик на гене в GenomeImporter.

**Оценка: 3 дня**

## Модуль 9: Sequence Editor (inline правки)

**Уже реализовано:** SequenceEditor.jsx с quick actions.

**Quick actions (один клик):**
- +TAA / +TAG / +TGA стоп-кодон
- +ATG стартовый кодон
- +Kozak (GCCACC перед ATG)
- +His6 (C-конец или N-конец)
- Удалить стоп-кодон
- Удалить первый/последний нуклеотид

**Actionable warnings:** ⚠ "нет стоп-кодона" → кнопка [+TAA] прямо в предупреждении.
**Live translation:** при редактировании ДНК — белок обновляется в реальном времени.
**Кодоны центрированы:** аминокислота точно под своим кодоном.

**Оценка: реализовано, полировка 0.5 дня**

## Модуль 10: Primer phosphorylation

**Проблема:** Для KLD мутагенеза и blunt-end лигирования нужны 5'-фосфорилированные праймеры. Студент забывает указать при заказе.

**Решение:**
- Флаг phosphorylated на каждом праймере
- Авто-установка для KLD и blunt-end лигирования
- Ручной toggle для остальных
- Бейдж [Ⓟ 5'-PO₄] в Primer Panel и Oligo Manager
- В TSV экспорте: колонка "Модификация: 5'-Phosphorylation"
- Напоминание в Protocol Tracker

**Оценка: 0.5 дня**

## Модуль 11: Junction calcMode (длина vs Tm)

**Проблема:** Непонятно почему overlap 30 расширился до 35. Скрытая логика.

**Решение:** Два явных режима на каждом junction:
- "По длине" → задал 30 п.н., получил ровно 30 (Tm информационно)
- "По Tm" → задал 62°C, движок подобрал нужное количество п.н.

Toggle в junction config popup. По умолчанию: "По длине".

**Оценка: 0.5 дня**

## Модуль 12: Auto-adjust junction для фрагментов без ПЦР

**Проблема:** Фрагмент "из морозилки" (без ПЦР) не имеет праймеров. Overlap 50/50 невозможен.

**Решение:** Автоматическое переключение:
- A(ПЦР) + B(без ПЦР) → режим ◀ (весь overlap на rev_A)
- A(без ПЦР) + B(ПЦР) → режим ▶ (весь overlap на fwd_B)
- A(без ПЦР) + B(без ПЦР) → ⚠ предупреждение

Недоступные режимы серые в junction popup.

**Оценка: 0.5 дня**

## Модуль 13: Слияние фрагментов после завершения сборки

**Проблема:** После overlap PCR на canvas всё ещё 3 отдельных блока. Непонятно что "сборка завершена".

**Решение:** Кнопка "Готово" в Protocol Tracker → фрагменты сливаются в один полосатый блок:

```
[PglaA═══════GFP═══════TtrpC]
  зелёный    голубой    оранжевый
         2,078 п.н. (45 нг/µл) ✓
```

Toggle "🔍 Развернуть" / "📦 Свернуть" для просмотра исходных фрагментов.
Продукт автоматически в inventory + палитра для следующей сборки.

**Оценка: 1 день**
